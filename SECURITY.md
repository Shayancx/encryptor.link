# Security Policy

## ⚠️ Heads Up

This project — **encryptor.link** — is something I (@Shayancx) built to sharpen my skills with Ruby and Ruby on Rails. It’s a personal learning project, not a commercial-grade service. I care deeply about privacy and security, but I’m not a professional security researcher. So: use at your own risk, and keep in mind that this is a work-in-progress.

---

## 🔒 Security Basics

**encryptor.link** is meant to be a privacy-respecting, zero-knowledge, end-to-end encrypted messaging tool. Everything you send is encrypted *before* it ever touches the server — that’s the core idea. Still, since I’m learning as I go, it might not be bulletproof.

---

## 🕵️‍♀️ Found a Security Issue?

Please don’t post it publicly.

Instead, open a [GitHub Security Advisory](https://github.com/Shayancx/encryptor.link/security/advisories). It stays private and lets us work together to fix the problem.

When you do, please include:

* What you found and how you found it
* Steps to reproduce the issue
* What kind of risk it poses
* Any ideas you have for fixing or mitigating it

**What I’ll do:**

* I’ll reply within 48 hours
* I’ll keep you in the loop about progress
* I’ll fix the issue as soon as I can
* I’ll credit you (unless you’d rather stay anonymous)

---

## 🔐 Security Features at a Glance

* **End-to-End Encryption**
  Messages and files are encrypted in the browser. Nothing leaves your device unencrypted.

* **No Accounts**
  You don’t need to sign up. Less data means less to lose.

* **Self-Destructing Messages**
  Messages delete themselves after being read. Nothing sticks around.

---

## 🧠 Tips for Using encryptor.link Safely

* **Be careful how you share links**
  Use secure channels. Don’t post links in public spaces.

* **Keep your browser updated**
  And always double-check the domain: you're only safe on **encryptor.link**.

* **Think before you share**
  Avoid sending anything that could hurt you or someone else if leaked. Recipients can still copy/save content.
